Library Class

The Library class is the class that gets returned when you load the library. Unlike other classes, the Library Class does not inherit any methods or properties from the BaseElement class.

Parametersâ€‹

note

Unlike Other Classes' Parameters, The Library's Are Using getgenv() - This means the executor you're using must support it as it is not built into Roblox itself. However, none of the Library's Parameters are required - so an executor having getgenv() isn't a must

getgenv().InterfaceName : stringâ€‹

An editable string that determines the Instance's Name upon creation.

This property is not required to be set, as Starlight has a built-in fallback system. However, it is highly recommended since it helps Starlight make sure your script is not affected by other using Starlight as well.

This property is editable, and can be changed. However, the Property is only checked on creation. Meaning after initialising the library, setting it or changing afterwards is reduntant This also leads to the must of putting it as the first thing in terms of Interface.

-- must be place uptop
getgenv().InterfaceName = "MyScript"
  
-- initialising of lib here
local Starlight = loadstring(game:HttpGet('https://raw.githubusercontent.com/Nebula-Softworks/Starlight-Interface-Suite/main/Source.lua'))()
